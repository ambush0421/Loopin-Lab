# Gap Analysis: Building Report Pro

## 1. 개요 (Overview)
- **분석 대상:** Building Report Pro 구현 코드 vs 설계 문서
- **분석 일시:** 2026.02.08
- **일치율:** 약 75%

## 2. 일치 항목 (Matched Features)
- **대시보드 레이아웃:** `DashboardShell`을 통한 3단 그리드(사이드바 + 메인 + 위젯) 구조가 정확히 구현됨.
- **핵심 지표 카드:** `TopMetrics` 컴포넌트가 설계대로 구현됨.
- **수익성 계산기:** `YieldCalculator`의 ROI 자동 계산 로직이 정상 구현됨.
- **지도 연동:** `InvestmentMap`을 통해 카카오맵이 연동됨 (API 키 필요).

## 3. 불일치 항목 (Gaps Identified)

### 3.1 실거래가 데이터 매칭 미구현 (Critical)
- **설계:** `api/market-price` 호출을 통해 주변 반경 500m 실거래가 데이터를 가져오기로 함.
- **구현:** 현재 해당 API Route가 없으며, 프론트엔드에서도 호출 로직이 없음.
- **영향:** 지도에 마커가 표시되지 않고, 시세 비교 차트를 그릴 데이터가 없음.

### 3.2 시세 차트 컴포넌트 누락 (Major)
- **설계:** `MarketChart` 컴포넌트를 통해 평단가 추이를 시각화하기로 함.
- **구현:** 파일 자체가 생성되지 않음.
- **영향:** '모던 대시보드'의 핵심인 시각적 분석 요소가 부족함.

### 3.3 히스토리 기능의 제한적 구현 (Minor)
- **설계:** 로컬 스토리지 저장 및 불러오기.
- **구현:** UI(`HistorySidebar`)만 더미 데이터로 구현됨. 실제 저장 로직 없음.

## 4. 권장 개선 사항 (Recommendations)

### 4.1 1차 개선 (Iterate Phase 1)
- **MarketChart 컴포넌트 추가:** Recharts를 활용하여 연도별 평단가 추이 차트 구현.
- **Mock Data 적용:** 실거래가 API 개발 전까지, 데모용 더미 데이터를 사용하여 차트와 지도 마커를 먼저 활성화하여 UX 완성도 제고.

### 4.2 2차 개선 (Iterate Phase 2)
- **실거래가 API 개발:** 국토부 아파트/상업업무용 실거래가 API 연동 (백엔드).
