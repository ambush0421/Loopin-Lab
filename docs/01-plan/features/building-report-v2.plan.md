# Plan: Building Report Pro v2 (사용자 피드백 기반 고도화)

> 버전: 2.0.0 | 작성일: 2026-02-10 | 상태: 초안(Draft)

## 1. 개요 (Executive Summary)
v1.0.0의 성공적인 런칭 이후, 실무 사용자들의 피드백을 반영하여 **데이터 저장**, **모바일 편의성**, **추가 정보 제공**을 중심으로 한 v2.0.0 업그레이드를 계획합니다.

## 2. 주요 피드백 및 개선 사항 (Feedback & Improvements)
| 구분 | 사용자 피드백 | 개선 방안 |
|------|--------------|-----------|
| **데이터 보관** | "만든 보고서를 저장해두고 나중에 다시 보고 싶어요." | **DB 연동:** 로그인 및 보고서 클라우드 저장 기능 구현. |
| **정보 부족** | "공시지가랑 토지 정보도 같이 나왔으면 좋겠어요." | **추가 API:** 공시지가 및 토지대장 API 추가 연동. |
| **사용성** | "폰으로 현장에서 바로 검색하고 싶어요." | **모바일 최적화:** 반응형 UI 개선 및 모바일 전용 뷰 추가. |

## 3. 기능 명세 (Feature Specifications)
### 3.1. 사용자 시스템 (Auth)
- 이메일/소셜 로그인 (NextAuth.js).
- 사용자별 마이페이지 (내 보고서 목록).

### 3.2. 데이터베이스 (Database)
- **Tech:** Supabase (PostgreSQL).
- **Schema:** `Users`, `Reports` 테이블 설계.
- JSON 형태로 API 응답 원본 및 분석 코멘트 저장.

### 3.3. 확장 데이터 (Extended Data)
- **공시지가 API:** 최근 5년치 공시지가 추이 그래프 제공.
- **토지대장 API:** 지목, 토지이동연혁 등 토지 상세 정보 제공.

## 4. 성공 기준 (Success Criteria)
- 모바일 환경에서의 보고서 가독성 100% 확보.
- 보고서 저장 및 불러오기 로딩 시간 1초 이내.
- 공시지가 데이터 정확도 100%.

## 5. 일정 (Timeline)
- **Plan:** 오늘 (완료)
- **Design:** 내일 (DB 스키마 및 UI 설계)
- **Do:** 이번 주 내 (기능 구현)
- **Check:** 다음 주 초 (사용자 테스트)
